Features

🔌

Real-time WebSockets

All four players see live updates the moment any player submits their order — no page refresh needed.

🤖

AI Fallback

Any unfilled role is taken over by a pipeline-aware base-stock AI, so the game always advances.

📊

Analytics & Charts

End-of-game results page shows per-role costs, service level, Bullwhip Effect Index, and full Chart.js history.

📋

Instructor View

Live game overview for the session creator, plus CSV export for post-session analysis.

⏱️

Classic MIT Mechanics

2-week order delay + 2-week shipment delay, $0.50 holding / $1.00 backlog costs, classic demand schedule.

🔒

Information Hiding

Each player can only see their own inventory and pipeline — the core source of the bullwhip effect.

Deploy the Full Game in Minutes

Fork or connect the repo to Render

Go to render.com, sign in, then click New → Blueprint and select this repository. Render will detect render.yaml automatically.

Set environment variables

Render auto-generates SECRET_KEY, DATABASE_URL, and REDIS_URL from the blueprint. Add ALLOWED_HOSTS and CSRF_TRUSTED_ORIGINS with your .onrender.com hostname once the service is created.

Click "Apply" — that's it

Render builds the image, runs collectstatic + migrate, and starts Daphne. Your game is live at https://<your-service>.onrender.com.

Update this landing page

Edit the href on the Play the Full Game button in index.html to point to your live URL, then push to main.

Prefer a different host? Railway and Fly.io also work — both support ASGI + WebSockets + Redis. Use the Procfile inside beer11C/.
